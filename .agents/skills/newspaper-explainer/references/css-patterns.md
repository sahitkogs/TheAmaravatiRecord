Reusable CSS class vocabulary for broadsheet newspaper pages. Read this before generating to understand what components exist, when to use each, and what constraints apply. The canonical template at ./templates/broadsheet.html demonstrates every pattern in context.

[!PALETTE]: Four CSS variable tiers define the entire visual language: a) PAPER variables (--paper, --paper-tinted) for backgrounds, b) INK variables (--ink, --ink-mid, --ink-light) for text at three weight levels, c) RULE variables (--rule, --rule-mid, --rule-light) for lines and borders at three intensities, d) ACCENT variables (--red, --green, --blue, --amber plus their -bg variants) used ONLY in badges, dot indicators, and dispatch card labels. Both light and dark schemes are defined via prefers-color-scheme media query. NEVER EVER add new color variables or use raw hex values outside this palette.

[!TYPOGRAPHY]: Four font tiers are loaded via Google Fonts CDN: a) --font-display (Playfair Display) for the masthead title and all headlines (lead, sidebar, story), b) --font-body (Source Serif 4) for article prose, letter bodies, widget bodies, and pull quotes, c) --font-sans (system sans-serif stack) for ALL metadata: category labels, badges, bylines, widget labels, ticker, dispatch card labels, masthead meta, colophon, d) --font-mono (JetBrains Mono) for inline code references only. Inline code gets font-size 0.88em with a paper-tinted background and 1px 4px padding.

[!MASTHEAD]: The .masthead container is text-align center with a 3px solid top border. It contains three elements: a) .masthead__meta as a flex row with space-between holding volume/issue (left), date (center), edition info (right) — 9px uppercase tracked sans, b) .masthead__title in display font at clamp(48px, 8vw, 72px) weight 900 with a 1px top border, c) .masthead__tagline in italic display font at 12px with a 3px bottom border. The masthead borders frame the newspaper identity.

[!TICKER]: The .ticker container uses a dark ink background with paper-colored text, 9.5px uppercase tracked sans. Inside, a .ticker__track div with display inline-flex runs a CSS keyframe animation (ticker-scroll, translateX 0 to -50%, 35s linear infinite). The track content is DUPLICATED to create seamless looping. Items are separated by .ticker__sep elements (diamond character in --red). The ticker pauses on hover via animation-play-state. In dark mode, the ticker switches to paper-tinted background with ink text and a 1px border.

[!BADGES]: The .badge class is an inline-block with 9px uppercase tracked sans, 2px 8px padding. Six variants exist: a) .badge--progress (red background, white text) for active/in-progress states, b) .badge--review (blue background, white text) for items under review, c) .badge--nominal (green background, white text) for healthy/complete states, d) .badge--advisory (amber-bg background, amber text, amber border) for warnings, e) .badge--response (transparent background, red text, red border) for items needing response, f) .badge--queued (ink-mid background, paper text) for queued items. Use sparingly — one badge per article maximum.

[!CATEGORY-AND-BYLINE]: The .category class is 9px uppercase sans with 2px letter-spacing in ink-mid, placed above headlines. The .byline class is 9px uppercase sans with 1px letter-spacing in ink-light, placed below headlines. Byline fields are separated by pipe characters with ensp spacing. Every article carries both a category and a byline.

[!LEAD-STORY]: The .lead article occupies the left column of the .front-page grid (grid-template-columns: 1fr 300px) with a right border and 24px right padding. It contains: a) .lead__headline in display font at clamp(28px, 5vw, 40px) weight 900, b) .lead__deck in display font at 16px weight 400 in ink-mid for the subheadline, c) a .drop-cap paragraph whose ::first-letter floats left at 4.2em display font weight 900, d) an optional .inset-box floating right at 210px width with 1.5px border containing .inset-box__title (10px uppercase sans) and .inset-box__detail (12.5px body in ink-mid), e) a .pull-quote blockquote in italic display font at 16px with 2px top border and 1px bottom border, f) a .column-text div with column-count 2, 24px gap, and a 1px column-rule. The drop cap and inset box coexist via float layout — the drop cap paragraph wraps around the floating inset box.

[!PROGRESS-BAR]: The .progress-bar is a 10px high container with paper background and 1px ink border. Its child .progress-bar__fill uses a repeating-linear-gradient at -45deg creating diagonal hatching in ink (3px ink, 3px transparent). Width is set inline as a percentage. Below it, .progress-bar__label is a flex row with space-between showing 0%, current value, and 100% in 8.5px uppercase tracked sans.

[!SIDEBAR]: The .sidebar occupies the right column with 24px left padding. It contains .sidebar__widget containers (1px border, 14px padding, 16px bottom margin) with .widget__label headers (9px uppercase tracked sans in ink-mid) and .widget__body content (13px body). The .agents-table uses 12px text with dot indicators (.dot at 7px with --active/--busy/--idle color variants) and right-aligned monospace values. Sidebar articles (.sidebar__article) are separated by 1px top borders with .sidebar__headline in 17px display weight 700 and .sidebar__body in 13px.

[!STORIES-ROW]: The .stories-row is a 3-column grid (repeat(3, 1fr)) with no gap. Column separation uses 1px left borders and 20px horizontal padding on each .story (first child has no left border or left padding). Each story contains .story__headline in 18px display weight 700 and .story__body in 13.5px.

[!LOWER-FOLD]: The .lower-fold is a 2-column grid (1fr 320px) with a 3px top border. The .letters section occupies the left column with 24px right padding and a right border. It uses .letters__super (9px uppercase tracked sans) and .letters__title (20px italic display weight 700 with 1.5px bottom border). Individual .letter blocks alternate between .letter--question (italic body, representing incoming questions) and regular letters (representing responses). The .dispatch-board occupies the right column with 24px left padding. Its .dispatch-board__title is centered 12px uppercase sans with 3px letter-spacing flanked by diamond characters. Individual .dispatch-card containers have 1px borders and 12px padding with .dispatch-card__label headers colored by type (--wanted in red, --notice in blue, --review in amber, --recovery in green, --merge in red) and .dispatch-card__body in 12px.

[!COLOPHON]: The .colophon footer is a flex row with space-between in 9px uppercase tracked sans in ink-light, topped by a 3px border.

[!OVERFLOW]: All grid and flex children need min-width 0 to allow shrinking. Use overflow-wrap break-word on text containers. NEVER EVER use display flex on li elements for marker positioning — it creates anonymous flex items that cannot shrink, causing overflow with inline code badges. Use absolute positioning for list markers instead. Tables with wide content need overflow-x auto on their container.

[!RESPONSIVE]: At max-width 800px: a) all grids collapse to single column (front-page, stories-row, lower-fold), b) column separating borders switch from vertical to horizontal, c) column-text reverts to column-count 1, d) inset-box becomes full width with no float, e) colophon stacks vertically centered. Padding reduces from 40px to 20px.

[!DARK-MODE]: The dark scheme inverts paper to near-black (#1a1815) and ink to warm light (#e8e4dd). Rules become lighter (#706c65). Accent colors shift to brighter variants for visibility on dark backgrounds. The ticker switches from ink-background to paper-tinted-background with a subtle border. Badge colors auto-adjust via the CSS variables.

[!PRINT]: Print styles set white background, simplified ticker, hidden submit sections, and simplified badges. The broadsheet container goes full width with no padding.
